This legacy page has been superseded by the architecture docs:

- `platform/architecture/routing`
- `platform/architecture/workspaces-permissions`

The short version is:

- `apps/web/src/proxy.ts` is the current edge/router coordinator for workspace-aware URL handling.
- `personal` and `internal` are canonical browser slugs; database queries should still use UUID workspace IDs.
- `WorkspaceWrapper` is the server-page entry point for workspace resolution.
- `normalizeWorkspaceId()` is the route-handler entry point for workspace-scoped params.
- `getPermissions()` is the fine-grained authorization helper layered on top of routing and membership.
- Root redirects can be user-configured per workspace via `user_workspace_configs` (config id: `ROOT_DEFAULT_NAVIGATION`). Redirect targets should be validated at request time and stale deep links (for example deleted task boards) must gracefully fall back to a broader route.

Prefer the two architecture pages above for current implementation details.
