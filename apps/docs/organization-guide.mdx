## üìù Adding New Documentation

### 1. Choose the Right Section

| Content Type | Section | Example |
|--------------|---------|---------|
| UI Components | Components | `components/button.mdx` |
| Development Guides | Development Tools | `development-tools/testing.mdx` |
| Code Samples | Examples | `examples/button-example.tsx` |
| Platform Features | Features | `features/calendar.mdx` |
| AI Features | AI Features | `ai-structured-data.mdx` |
| User Customization | Personalization | `personalization/themes.mdx` |
| API Documentation | API References | `api-reference/endpoint/create.mdx` |
| Beta Features | Experiments | `experiments/new-feature.mdx` |

### 2. Create the File

```mdx
---
title: "Your Page Title"
description: "Brief description"
updatedAt: "2024-12-19"
---

# Your Page Title

Content goes here...
```

### 3. Update Navigation

Add to `mint.json` in the appropriate section:

```json
{
  "group": "Components",
  "pages": [
    "components/index",
    "components/workspace-wrapper",
    "components/your-new-component"
  ]
}
```

### 4. Update Index Page

Add a link to the new page in the section's `index.mdx`:

```mdx
### Your New Component

Description of what it does and how to use it.

- [Your New Component](./your-new-component) - Complete documentation
```
